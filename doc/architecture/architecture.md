## 系统架构

DeepResearch 采用模块化架构设计，将系统分为多个独立的模块，每个模块负责特定的功能。系统的核心架构包括以下几个部分：

### 1. 核心模块

- **LLM 模块**：负责与大语言模型的交互，包括模型实例的创建、缓存和调用。
- **Prompt 模板模块**：负责加载和应用 prompt 模板，生成格式化的消息。
- **Agent 模块**：负责构建和运行智能研究工作流，包括任务规划、工具调用、评估与迭代。
- **工具模块**：提供各种工具，如搜索工具、评估工具等，支持 agent 的工作流程。
- **配置模块**：负责加载和管理系统配置，包括 LLM 配置、搜索工具配置等。

### 2. 依赖关系

```{mermaid}

    A[Agent 模块] --> B[LLM 模块]
    A --> C[Prompt 模板模块]
    A --> D[工具模块]

    C --> F[配置模块]
```

## 模块关系

### 1. LLM 模块

LLM 模块是系统的核心组件之一，负责与大语言模型的交互。它提供了以下功能：

- 管理 LLM 实例的创建和缓存
- 支持流式和非流式响应
- 处理 LLM 响应的格式化

LLM 模块依赖于配置模块，从配置文件中加载 LLM 的配置信息。

### 2. Prompt 模板模块

Prompt 模板模块负责加载和应用 prompt 模板，生成格式化的消息。它提供了以下功能：

- 加载所有 prompt 模板
- 支持懒加载，提高系统启动速度
- 应用模板，注入变量

Prompt 模板模块依赖于配置模块，从配置文件中加载模板信息。

### 3. Agent 模块

Agent 模块是系统的核心组件之一，负责构建和运行智能研究工作流。它提供了以下功能：

- 构建 agent 图，定义工作流程
- 执行工作流程，包括任务规划、工具调用、评估与迭代
- 管理工作流程的状态和配置

Agent 模块依赖于 LLM 模块、Prompt 模板模块和工具模块，使用这些模块提供的功能来完成工作流程。

### 4. 工具模块

工具模块提供各种工具，支持 agent 的工作流程。它提供了以下功能：

- 搜索工具：使用搜索引擎搜索信息
- 评估工具：评估内容质量

工具模块依赖于配置模块，从配置文件中加载工具的配置信息。

### 5. 配置模块

配置模块负责加载和管理系统配置，包括 LLM 配置、搜索工具配置等。它提供了以下功能：

- 加载 TOML 配置文件
- 解析配置信息
- 提供配置访问接口

## 工作流程

DeepResearch 的工作流程采用"任务规划 → 工具调用 → 评估与迭代"的模式，具体步骤如下：

1. **任务规划**：根据用户的查询，agent 规划研究任务，确定需要执行的步骤。
2. **工具调用**：agent 调用相应的工具，如搜索工具，获取相关信息。
3. **评估与迭代**：agent 评估获取的信息，判断是否满足要求，如果不满足，则迭代执行上述步骤，直到满足要求。
4. **结果生成**：agent 生成最终的研究结果，返回给用户。

### 工作流程示意图

```{mermaid}
flowchart TD
    A[用户输入查询] --> B[任务规划]
    B --> C[工具调用]
    C --> D[评估与迭代]
    D --> E{是否满足要求?}
    E -->|否| B
    E -->|是| F[结果生成]
    F --> G[返回结果给用户]
```

## 数据流程

1. **输入数据**：用户的查询被转换为消息对象，传递给 agent。
2. **处理数据**：agent 处理输入数据，调用相应的模块和工具。
3. **输出数据**：agent 生成最终的研究结果，返回给用户。

### 数据流程示意图

```{mermaid}
flowchart TD
    A[用户输入] --> B[消息对象]
    B --> C[Agent 模块]
    C --> D[LLM 模块]
    C --> E[工具模块]
    D --> F[LLM 响应]
    E --> G[工具结果]
    F --> H[结果处理]
    G --> H
    H --> I[最终结果]
    I --> J[返回给用户]
```

## 性能优化

DeepResearch 采用了以下性能优化措施：

1. **LLM 实例缓存**：使用 LRU 缓存管理 LLM 实例，避免重复创建实例。
2. **LLM 响应缓存**：缓存 LLM 的响应，避免重复调用 LLM。
3. **Prompt 模板懒加载**：懒加载 prompt 模板，提高系统启动速度。
4. **并行处理**：在可能的情况下，使用并行处理提高系统性能。

## 扩展性

DeepResearch 的架构设计具有良好的扩展性，支持以下扩展方式：

1. **添加新的 LLM**：通过配置文件添加新的 LLM 配置。
2. **添加新的工具**：通过实现工具接口，添加新的工具。
3. **添加新的 prompt 模板**：通过添加新的模板文件，添加新的 prompt 模板。
4. **扩展工作流程**：通过修改 agent 图，扩展工作流程。

## 部署架构

DeepResearch 支持本地部署，部署架构如下：

1. **配置文件**：存储系统配置，包括 LLM 配置、搜索工具配置等。
2. **代码模块**：实现系统的核心功能。
3. **依赖库**：系统依赖的第三方库。
4. **用户数据**：存储用户的研究结果和配置。

```{mermaid}
    B --> C[配置文件]
    B --> D[代码模块]
    B --> E[依赖库]
    B --> F[用户数据]
```
